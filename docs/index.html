为什么需要这个项目？ 2026年3月31日，武汉近百辆萝卜快跑在高架路段同时熄火。乘客被困2小时，SOS无人应答，最终依靠交警逐车施救。三个月前，旧金山停电导致Waymo大面积瘫痪，应急热线等待2.5小时。当L4车队规模化运营时，这种靠人力逐车施救的模式不可持续。Why this project? On March 31, 2026, ~100 Baidu robotaxis shut down simultaneously on Wuhan's elevated highways. Passengers were trapped for 2 hours. SOS unanswered. Resolution: police rescuing vehicles one by one. Three months earlier, a San Francisco blackout left Waymo's fleet stranded, with emergency services waiting 2.5 hours on hold. At scale, this is unsustainable.

四个模块Four Modules

ROAM 提供四个相互关联的开源模块，构成L4远程运营的完整基线Four interconnected open-source modules forming a complete baseline for L4 remote operations

模块一Module 1

事故 / 异常事件库Incident Database

Incident Database

16+ 结构化事件记录，覆盖4家运营商（Waymo / Cruise / 百度Apollo / Pony.ai），YAML标准格式，社区持续更新16+ structured records across 4 operators, standardized YAML format, community-maintained

↳ 支撑GB标准附录D安全档案中的运营证据↳ Supports Safety Case evidence (GB Standard Appendix D)
模块二Module 2

场景分类体系Scenario Taxonomy

Scenario Taxonomy

6大类、26子场景。含严重度S0–S4分级、紧急度U0–U3分级，对齐ISO 34502场景分层方法6 categories, 26 sub-scenarios. Severity S0–S4, urgency U0–U3, aligned with ISO 34502

↳ 对齐GB标准表D.1整车危害分类↳ Aligned with GB Standard Table D.1 hazard categories
模块三Module 3

参考架构Reference Architecture

Reference Architecture

三层决策模型（AI自主 → 人工确认 → 远程遥控）。含API接口规范、场景-处置映射表、操作员界面指南Three-layer decision model (AI autonomous → human confirm → remote driving). Includes API spec and handler mapping

↳ 对齐GB标准附录C.2远程协助要求↳ Aligned with GB Standard Appendix C.2 remote assistance
模块四Module 4

评价基准Evaluation Benchmarks

Evaluation Benchmarks

8项KPI指标（MTTR、AI处置率、误升级率等），含目标值与自然驾驶基线数据8 KPIs (MTTR, AI resolution rate, false escalation rate, etc.) with target values and naturalistic driving baselines

↳ 支撑残余风险接受准则（碰撞率 <10⁻⁴/h）↳ Supports residual risk criteria (collision rate <10⁻⁴/h)

三层决策模型Three-Layer Decision Model

AI优先处置，人工逐级兜底AI-first response, human fallback by escalation

Layer 1 · AI自主处置AI Autonomous Response

70%

场景识别 → 风险评估 → 决策生成 → 自动执行，无需人工介入Scenario recognition → risk assessment → decision generation → autonomous execution

靠边停车 · 驶入服务区 · 低速绕障 · 感知模块重启 · 乘客语音安抚 · 调度备用车辆Pull over · enter service area · low-speed reroute · perception restart · passenger voice guidance · dispatch backup vehicle

↳ 对齐GB标准C.2.1.2：ADS应不依赖远程协助执行DDT↳ GB C.2.1.2: ADS shall not depend on remote assistance for DDT

Layer 2 · AI辅助 + 人工确认AI-Assisted + Human Confirm

25%

AI生成处置方案（含置信度）→ 远程安全员审核确认/修改 → 车辆执行AI generates action plan with confidence → remote operator reviews and confirms → vehicle executes

复杂路口脱困 · 交警手势解读 · 多车冲突裁决 · 报警/叫救护车决策 · 乘客异常行为Complex intersection escape · police gesture interpretation · multi-vehicle conflict · emergency call decision

↳ 对齐GB标准C.2.3：ADS应能够及时接收远程协助指令↳ GB C.2.3: ADS shall receive remote assistance commands in a timely manner

Layer 3 · 远程遥控 / 线下派人Remote Driving / On-Site Dispatch

5%

远程人员直接接管车辆控制（≤10km/h）或派遣线下救援团队到现场Remote operator takes direct vehicle control (≤10km/h) or dispatches on-site rescue team

车辆硬件故障 · 批量系统瘫痪 · 事故现场处理 · 车辆起火 · 乘客物理救援Hardware failure · mass system shutdown · accident scene · vehicle fire · passenger physical rescue

↳ 对齐德国StVFernLV远程驾驶框架 / Tesla Teleops模式↳ Aligned with German StVFernLV / Tesla Teleops model
