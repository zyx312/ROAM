## 为什么 ROAM 需要标准化？

截至 2026 年 4 月，全球没有任何一部 ISO/SAE/IEC 标准专门以 **"L4 自动驾驶远程运营与异常管理"** 为对象。所有现有相关标准都是 **"要求型"**（requirement specification）文档，没有一部是 **"参考架构型"**（reference framework / architecture）。这构成了 ROAM 的核心机遇：

> **ROAM 的战略定位不是再造一部 "requirement specification"，而是成为全球第一部 L4 远程运营与异常管理的"参考架构"标准。**

## 关键判断

### 判断 1：中国标准体系"四路并行，各自为政"

目前中国涉及 L4 远程运营的标准分属四个互不统属的体系：

| 体系 | 主管 | 代表标准 | 侧重 |
|------|------|---------|------|
| **通信口** | 工信部 / CCSA | YD/T 4778-4783 系列（2024） | 5G 信道与技术接口 |
| **汽车口** | 工信部 / TC114 | GB/T 44721、GB/T 45312、GB 44497、强制 GB 草案 | 车辆功能安全 |
| **交通口** | 交通部 / TC521 | 在研 GB/T《远程安全员技能要求》 | 人员与运营规范 |
| **网安口** | 工信部 / 网信办 | GB 44495、GB 44496 | 信息安全 |

**缺口**：没有一个整合性的"运营中心参考架构"标准贯穿四个体系。ROAM 可以填补此空白。

### 判断 2：国际上没有对应标准

- **ISO / SAE / IEC**：无一专门标准
- **SAE J3016**：明确**拒绝**定义 teleoperation（"consistently defined" 不足）
- **ISO 21448 / 34502 / 22737 / 23793**：均未将"远程运营中心"作为独立对象
- **UN GTR on ADS**：仍在 2026 年草案阶段
- **德国 StVFernLV / 英国 AV Act 2024**：均为法规而非技术标准，仅覆盖 remote driver 许可与责任

### 判断 3：T/CSAE 体系是 ROAM 的最佳切入点

**理由**：
1. **学术权威性高** + **国际认可度强**（已有 3 项 T/CSAE 标准转化为 ISO）
2. **无现有竞争标准**（经核实 T/CSAE 200-2021 是铝合金轮毂标准，不是远程驾驶标准 —— ROAM 不会与任何现有 CSAE 团标冲突）
3. **与作者背景契合**：吉林大学副教授 + DRIVEResearch + 卓驭联合的身份适合 T/CSAE 立项
4. **周期短**：12–18 月可发布，作为国标和国际标的实证基础

### 判断 4：紧迫的政策窗口期

**2026-04-13 截止**：工信部强制性 GB《智能网联汽车 自动驾驶系统安全要求》征求意见稿结束。这是将 ROAM 术语、架构、KPI 以**最低成本最高影响**的方式嵌入国标体系的唯一窗口。
